# TODO — идеи для улучшения translate-lint

## Новые правила

| Правило | Описание |
|---|---|
| **`placeholderConsistency`** | Проверяет что плейсхолдеры `{name}`, `{{count}}`, `%s` совпадают по составу во всех языках. Очень частая боль на проектах |
| **`sortKeys`** | Ключи в языковом файле должны быть отсортированы. Уменьшает merge-конфликты в командах |
| **`maxValueLength`** | Ограничение на длину текста перевода. Полезно для UI где длинный текст ломает верстку |
| **`noHtmlInValues`** | Запрещает или предупреждает об HTML-тегах в значениях (`<b>text</b>`) |

## Новые флаги / поведение

| Флаг | Описание |
|---|---|
| **`--watch`** | Перезапускать lint при изменении файлов. Как `eslint --watch` — удобно в dev |
| **`--output <file>`** | Сохранять результат в файл. Особенно полезно с `--format json` для CI-артефактов |
| **`--quiet`** | Показывать только ошибки, скрывать предупреждения (`--zk warning` становится невидимым) |
| **`--print-config`** | Вывести итоговый конфиг после мержа CLI + файл + defaults. Как `eslint --print-config` — для отладки |
| **`--no-color`** | Отключить ANSI-цвета в stylish выводе. Нужно для логов в CI без цветовой поддержки |
| **`--bail`** | Остановиться после первой найденной ошибки |
| **`--cache`** / **`--cache-file`** | Кэшировать результаты по хешам файлов. Сильно ускоряет в больших проектах |
